# 社区

已经有很多人在使用 Composer，其中不少人也在积极贡献。

## 贡献

如果你想为 Composer 做贡献，请阅读 [README](https://github.com/composer/composer) 和 [CONTRIBUTING](https://github.com/composer/composer/blob/main/.github/CONTRIBUTING.md) 文档。

最重要的指导原则如下：

> 所有代码贡献——包括拥有提交权限的人员的贡献——都必须通过拉取请求，并且需要核心开发人员批准后才能合并。这是为了确保所有代码都经过适当的审查。
>
> Fork 项目，创建功能分支，然后向我们发送拉取请求。
>
> 为了确保代码库的一致性，你应该确保代码遵循 [PSR-12 编码标准](https://www.php-fig.org/psr/psr-12/)。

## 支持

IRC 频道位于 irc.libera.chat: [#composer](ircs://irc.libera.chat:6697/composer)。

[Stack Overflow](https://stackoverflow.com/questions/tagged/composer-php) 和 [GitHub Discussions](https://github.com/composer/composer/discussions) 都有 Composer 相关问题的集合。

对于付费支持，我们通过聊天和邮件为 [Private Packagist](https://packagist.com) 客户提供 Composer 相关的支持。
